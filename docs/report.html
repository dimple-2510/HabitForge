    • Chapter 1: Introduction

    1.1 Project Overview

    HabitForge is a full-stack web application designed to help users build positive habits, break negative ones, and track their daily progress through an intuitive, gamified interface. The application leverages modern web technologies including Next.js 14, Supabase (PostgreSQL), and Tailwind CSS to deliver a responsive, secure, and feature-rich user experience.

    The project was developed as part of the MCA 4th Semester Major Project requirement (Subject Code: 23ONMCR-753, 12 Credits) at Chandigarh University's Centre for Distance & Online Education.

    1.2 Problem Statement

    Building and maintaining habits is a fundamental challenge in personal development. Research shows that it takes an average of 66 days to form a new habit (Lally et al., 2010). Yet most people struggle with:

    • Lack of consistency tracking — No easy way to see daily progress
    • No visual motivation — Missing streak counters and achievement systems
    • Poor analytics — Cannot identify patterns in habit completion
    • One-size-fits-all approach — Existing apps don't support diverse habit types

    HabitForge addresses these gaps by providing a comprehensive, type-aware habit tracking system with rich analytics and gamification.

    1.3 Objectives

    • Develop a full-stack web application for habit tracking with support for multiple habit types
    • Implement secure user authentication with email/password
    • Design an intuitive, mobile-first responsive interface
    • Build an analytics dashboard with charts, heatmaps, and habit strength metrics
    • Incorporate gamification through a badge/achievement system
    • Apply complete SDLC methodology from requirements through deployment
    • Document the project comprehensively for academic evaluation

    1.4 Scope

    Included:

    • User authentication (email/password)
    • CRUD operations for 4 habit types (positive, negative, target_count, groups)
    • Daily check-in system with optional journal notes
    • Streak tracking (current, longest, total completions)
    • Analytics dashboard with charts and heatmaps
    • Gamification with 6 milestone badges
    • Responsive design with dark/light mode
    • Deployment on Vercel

    Excluded:

    • Native mobile applications (iOS/Android)
    • Social features (friend groups, shared challenges)
    • Push notifications
    • Multi-language support

    5.7 Database Design

    The system uses 6 PostgreSQL tables with Row Level Security (RLS) policies:

    5.7.1 Table: habits

    ColumnTypeConstraintsDescription
    idUUIDPKUnique identifier
    user_idUUIDFK, NOT NULLOwner
    nameVARCHAR(100)NOT NULLHabit name
    habit_typeVARCHAR(20)CHECKType
    target_valueINTEGERNULLNumeric goal
    frequencyVARCHAR(20)CHECKSchedule
    categoryVARCHAR(50)DEFAULTCategory
    is_activeBOOLEANDEFAULT trueSoft delete

    5.7.2 Table: habit_logs

    ColumnTypeConstraintsDescription
    idUUIDPKUnique identifier
    habit_idUUIDFK, NOT NULLHabit reference
    user_idUUIDFK, NOT NULLOwner
    completed_dateDATENOT NULLDate of completion
    count_valueINTEGERNULLFor target_count
    noteTEXTNULLJournal entry

    5.8 Security Design

    Row Level Security (RLS) policies ensure users can only access their own data:

    • habits: auth.uid() = user_id (SELECT, INSERT, UPDATE, DELETE)
    • habit_groups: auth.uid() = user_id (SELECT, INSERT, UPDATE, DELETE)
    • habit_logs: auth.uid() = user_id (SELECT, INSERT, DELETE)
    • badges: authenticated (SELECT)
    • user_badges: auth.uid() = user_id (SELECT, INSERT)

    Chapter 6: Implementation

    6.1 Project Structure

    The project follows Next.js 14 App Router conventions:

    Figure: Project Directory Structure

    6.2 Key Implementation Details

    6.2.1 Authentication Flow

    The authentication system uses Supabase Auth with email/password. Session management is handled via middleware with cookie persistence (maxAge: 1 year).

    6.2.2 Habit CRUD Operations

    Habits support 4 types: positive, negative, target_count, and groups. Each type has specialized UI and tracking logic.

    6.2.3 Streak Calculation Engine

    The streak calculator analyzes consecutive days of habit completion, tracking current streak, longest streak, and total completions.

    6.2.4 Badge Auto-Award System

    Badges are automatically awarded via a PostgreSQL RPC function (award_badges()) that runs after each check-in.

    6.2.5 Analytics Dashboard

    The analytics page uses Recharts to display weekly bar charts, monthly area charts, category pie charts, and a yearly heatmap.

    6.2.6 Responsive Design

    Mobile-first approach with Tailwind CSS. Bottom tab bar on mobile, full top navigation on desktop. All inputs use minimum 16px font size.
